Product Description

This system is designed to efficiently manage and control classroom access using QR code technology.

  • QR Generation: Each user (student, teacher, etc.) gets a unique, personal QR code.
  • Entrance Scanning: A device at the classroom entrance is used to scan the QR code.
  • Access Logging: The system verifies the code and saves the exact date and time of entry into a database.
  • Log Visualization: Allows administrators to view a complete history of everyone who has entered.
×

Technical Specifications

The system is built with modern and robust technologies to ensure its reliability:

  • Programming Language: Python 3.x
  • Backend Framework: Flask
  • Frontend: HTML, CSS, JavaScript
  • Database: PostgreSQL
  • QR Library: qrcode (for Python)
  • Compatibility: Linux, Windows, and browsers like Chrome, Firefox, and Edge.
×

Main Functions

These are the most important actions the system can perform:

  • Generate QR Code: Creates a unique code for a user.
  • Scan QR Code: Reads and validates the code at the entrance.
  • Register Entry: Saves the access date and time to the database.
  • View Logs: Displays a detailed list of all registered entries.

Installation Checklist

For a developer to get the system running, they must follow these steps. Click each step to mark it as complete!

  • 1. Clone the repository from GitHub: git clone [URL]
  • 2. Install all necessary dependencies: pip install -r requirements.txt
  • 3. Configure the PostgreSQL database connection in config.py.
  • 4. Run the server to start the application: flask run

Common Troubleshooting

Problem: The QR code won't scan.

Possible Cause: Camera or reader issue.
Solution: Check the camera connection and restart the scanning device.

Problem: Cannot connect to the database.

Possible Cause: Incorrect configuration.
Solution: Verify that the connection details in config.py (user, password, host) are correct.

Problem: The server won't start.

Possible Cause: Missing dependencies.
Solution: Ensure you have run pip install -r requirements.txt in the correct virtual environment.

Frequently Asked Questions (FAQ)

Q: Can I use another database like MySQL or SQLite?

A: Yes, but you will need to change the connection code in the configuration and possibly install a new database connector.

Q: Can the system work without an internet connection?

A: Yes, as long as the server and database are hosted on a local network (LAN) that the scanning device can access.

Q: Is it possible to generate QR codes for many users at once?

A: Yes, the QR generation function can be modified to accept a list of users and create the codes in a batch.
